Web-Application-Design-Foundation

Jul 4, 2018 | Technique

Let us discuss the basic pipeline of designing a system

1/ What functional requirement do we need?

In other words, what kind of service we need? Usually read and write services are general requirement and we should seperate them. (The reason will illustrate below.)

2/ What is the non-functional requirement do we need?

Reliable? Avaibility? (CAP theorem) Low latency? Uploaded storage size limit?

3/ Computation:

3.0 Estimation for number of requests per day. Usually, Non popular(tinyURL, Pastebin): 1 million per day. Popular(Youtube, Twitter): over 5 million per day assuming 1 million daily active user and 5 requests per day. Also, estimation of read/write ratio.

3.1 Request per second for read and write. Totally, a day has 86400 seconds.

3.2 Storage estimation. Based on different content:

1/ Video average 10MB;

2/ Plain Text long: 10KB; short: exact calculation;

3/ Image: 500KB. With an expiration period constraint and how much we need for a day -> how much storages do we need totally?

3.3. Based on the size of content and request per second, estimate the bandwidth. How much do we need to transfer per second?

3.4 With the 80/20 principle (80% of the requests can be satisfied with 20% content.) We use the 20% of total storage to estimate the memory we need.

4/ API design.

For application server, what kinds of API should we have.

5/ Meta Data database schema design.

What tables do we need to have? What should be the tables’ schema?

6/ High level design.

What components do we have? Draw a baseline model first(no LBs, no horionzal scale first). Just a baseline model.

7/ Detailed explanation.

How we implement the read/write service with such design. How each component cooperate with each other.

8/ Optimizations. Usually directions:

8.1 Cache (Global Cache in two different ways or Distributed Cache);

8.2 Load Balancer;

8.3 Redundancy;

8.4 Data Partition or Sharding(Range based’s pros and cons / Hash even using Consistent Hash.)

For detailed knowledge for each part. Please turn to this blog: blog.csdn.net/firehotest. This is the chinese blog of mine.

Thanks for reading.
